Requirements
  • macOS 12 Monterey or later
  • Python 3.10+ Already present with Xcode Command Line Tools or install via brew install python
  • Claude Code installed and running Install from npm install -g @anthropic-ai/claude-code
  • Windows HUD running on the same network See setup-windows.html if you haven't set that up yet
Setup Steps
1
Copy the hook file

Copy hooks/remote_hook.py from the Windows PC to your Mac. You can use a USB drive, a shared network folder, or AirDrop. Place it at the path below:

mkdir -p ~/.claude/hud
# copy remote_hook.py from Windows share or USB to:
~/.claude/hud/remote_hook.py
2
Set your Windows IP

On the Windows PC, open a Command Prompt and run ipconfig. Look for the IPv4 Address under your active network adapter — it will look like 192.168.x.x.

Then on your Mac, export these environment variables (replace the IP with your Windows machine's address):

export CLAUDE_HUD_URL="http://192.168.1.10:51790"
export CLAUDE_HUD_DEVICE_ID="mac"
export CLAUDE_HUD_DEVICE_NAME="MacBook"
💡 Add these exports to ~/.zshrc so they persist across terminal sessions: 
echo 'export CLAUDE_HUD_URL="http://192.168.1.10:51790"' >> ~/.zshrc
echo 'export CLAUDE_HUD_DEVICE_ID="mac"' >> ~/.zshrc
echo 'export CLAUDE_HUD_DEVICE_NAME="MacBook"' >> ~/.zshrc
source ~/.zshrc
3
Test the connection

Send a test ping to the Windows HUD to confirm the network path is open:

curl -X POST http://192.168.1.10:51790 \
  -H "Content-Type: application/json" \
  -d '{"device_id":"mac","device_name":"MacBook","session_id":"test","state":"idle","event":"test"}'
Expected response: ok

If you get connection refused — check Windows Firewall (see the Troubleshooting section below or the Windows setup page).

4
Install the Claude Code hooks

Run this one-liner in your terminal. It writes the Claude Code hook entries into ~/.claude/settings.json and automatically backs up the existing file first.

python3 - <<'EOF'
import os, json, shutil, time, sys

SETTINGS = os.path.expanduser("~/.claude/settings.json")
HOOK = os.path.expanduser("~/.claude/hud/remote_hook.py")
CMD = f"python3 \"{HOOK}\""
MARKER = "claude-hud-remote"
EVENTS = ["SessionStart","UserPromptSubmit","PreToolUse","PostToolUse",
          "Notification","Stop","SubagentStop","SessionEnd"]
TOOL_EVENTS = {"PreToolUse","PostToolUse"}

cfg = {}
if os.path.exists(SETTINGS):
    shutil.copy2(SETTINGS, f"{SETTINGS}.bak-{time.strftime('%Y%m%d-%H%M%S')}")
    with open(SETTINGS) as f: cfg = json.load(f)

hooks = cfg.setdefault("hooks", {})
for ev in EVENTS:
    entry = {"hooks": [{"type": "command", "command": CMD}]}
    if ev in TOOL_EVENTS: entry["matcher"] = "*"
    hooks.setdefault(ev, []).append(entry)

with open(SETTINGS, "w") as f: json.dump(cfg, f, indent=2)
print("Hooks installed into", SETTINGS)
EOF
5
Verify

Start a new Claude Code session on your Mac. Within a second you should see the Windows HUD overlay update.

Green dot — Mac session is idle
Amber pulsing — Mac is working
Red — Mac needs your approval

You can also trigger the hook manually to test without starting a full session:

python3 ~/.claude/hud/remote_hook.py <<< '{"hook_event_name":"UserPromptSubmit","session_id":"test123"}'
✓ If the HUD overlay on Windows changes colour, everything is working correctly.
Primary Device
On the Windows PC, right-click the HUD overlay and select Primary device, then choose MacBook. This makes Mac alerts take top billing in the overlay — so when your Mac needs attention, the indicator is always front and centre regardless of what other devices are doing.
Troubleshooting
curl returns connection refused

Windows Firewall is blocking port 51790. On the Windows PC, open an elevated Command Prompt (run as Administrator) and run:

netsh advfirewall firewall add rule name="Claude HUD" dir=in action=allow protocol=TCP localport=51790

Then retry the curl test from Step 3. Also verify the Windows HUD is actually running — look for the tray icon in the system clock area.

Hook runs but nothing shows on Windows

Check that the environment variable is set correctly in your current shell:

echo $CLAUDE_HUD_URL

If it prints nothing, your ~/.zshrc exports haven't been sourced. Run source ~/.zshrc and try again.

Also confirm the Windows HUD process is running — its tray icon should be visible in the taskbar notification area. If it isn't, restart the HUD from the Windows setup.

Permission denied running remote_hook.py

Make the hook script executable:

chmod +x ~/.claude/hud/remote_hook.py

Then re-run the manual test from Step 5 to confirm it works.
