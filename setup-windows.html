Step-by-step install

1
Download / clone the project

Clone with Git:

git clone https://github.com/your-username/claude-status-hud.git
cd claude-status-hud
or

Download a ZIP archive instead:

⬇ Download ZIP

Extract the ZIP to a folder of your choice, then open a terminal inside it.

2
Run the installer

This installs the Claude Code hooks and the required Python packages (pystray, Pillow):

cd claude-status-hud
python bin\install.py

The installer will print a confirmation when it finishes. If you see any errors, check the Troubleshooting section below.

3
Create the taskbar shortcut

Run this from a PowerShell prompt inside the project folder:

powershell -ExecutionPolicy Bypass -File create-shortcut.ps1

A "Claude Traffic Light" shortcut will appear on your Desktop. To pin it to the taskbar:

Right-click "Claude Traffic Light" on the Desktop → Pin to taskbar

4
Start the HUD

Double-click the taskbar shortcut, or launch directly from the terminal:

start-hud.cmd

A small floating traffic-light window will appear. It stays on top of all other windows and reflects your Claude Code session state.

5
Enable run on startup (optional)

To have the HUD start automatically when you log in, use either method:

Right-click the HUD overlay → Run on Startup ✓

or

Right-click the system tray icon → Run on Startup ✓

A checkmark confirms the option is enabled. To disable it later, click the same item again.

Multi-device — receiving from Mac

When the HUD starts it automatically opens a listener on port 51790. Your Mac sends Claude Code session state to this port over your local WiFi — no extra configuration needed on the Windows side.

To find your Windows machine's local IP address, open a terminal and run:

ipconfig

Look for the IPv4 Address under your active network adapter (e.g. 192.168.1.42). It typically starts with 192.168. or 10.

Share this IP with your Mac setup. The Mac will send state to: http://YOUR_IP:51790

Primary device toggle

When you run the HUD on multiple machines, you can designate one as the primary device. Alerts from non-primary devices are shown with the device name prefixed so you know which machine needs attention.

To change which machine is primary:

Right-click the overlay → Primary device → select the machine you want to treat as primary.

Example — when your Mac (non-primary) raises an alert, the overlay shows:

MacBook: Needs your approval

This keeps the overlay uncluttered: alerts from your current machine show as normal, while remote alerts always carry the device label.

Troubleshooting

HUD doesn't start

Verify that Python is installed and available on your PATH:

python --version

If the command is not recognised, download Python 3.10+ from python.org/downloads and make sure to tick "Add Python to PATH" during installation. Then restart your terminal and try again.

Tray icon missing

The tray icon requires the pystray and Pillow packages. Install them manually if the installer didn't catch them:

pip install pystray pillow

After installation, restart the HUD. If the icon still doesn't appear, check that your system tray is not set to hide all icons (Settings → Taskbar → Other system tray icons).

Not receiving Mac sessions

Windows Firewall may be blocking inbound connections on port 51790. Run the following command in an elevated (Administrator) PowerShell or Command Prompt to add a firewall rule:

netsh advfirewall firewall add rule name="Claude HUD" dir=in action=allow protocol=TCP localport=51790

Then confirm the HUD is running, check your Windows IP with ipconfig, and ensure that IP is entered correctly in the Mac setup.
